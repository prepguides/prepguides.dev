What is a Trie?

A Trie (pronounced "try") is a tree-like data structure that stores strings in a way that makes prefix-based operations very efficient. Each node represents a character, and paths from root to leaf nodes represent complete words.

Key Properties:

  • • Root node is empty (represents start of all words)
  • • Each path from root represents a prefix
  • • Leaf nodes (or marked nodes) represent end of words
  • • Common prefixes share the same path
Operation Time Complexity Space Complexity
Insert O(m) O(m)
Search O(m) O(1)
Delete O(m) O(1)

Where m is the length of the word

Common Interview Questions

  • Implement a Trie with insert, search, and startsWith operations. Classic trie implementation with boolean flags for end-of-word.
  • Find all words with a given prefix. Use DFS from the prefix node to find all complete words.
  • Design a search autocomplete system. Use trie for prefix matching and return top K suggestions.
  • Implement word search in a 2D grid. Use trie to store dictionary words and DFS in grid.
  • Design a spell checker. Use trie for fast word lookup and suggest corrections for typos.
